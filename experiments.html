Why MathML Couldn't Be Used Until Recently (Extra Credit)

Despite being part of the HTML specification for a long time, MathML had limited practical use until recently due to poor browser support. According to caniuse.com, the main issues were:

  • Chrome/Chromium browsers removed MathML support in 2013 (Chrome 25) and only re-added it in Chrome 109 (January 2023), leaving a nearly 10-year gap where the most popular browser didn't support MathML at all.
  • Safari had partial support but with many rendering inconsistencies and bugs that made it unreliable for production use.
  • Internet Explorer and early Edge had no support for MathML, forcing developers to use JavaScript polyfills like MathJax or KaTeX instead.
  • The lack of consistent cross-browser support meant developers couldn't rely on native MathML and had to use alternative solutions, creating a chicken-and-egg problem where browsers didn't prioritize MathML because developers weren't using it.

Only in 2023, with Chrome 109's release and improved support in Firefox and Safari, did MathML become viable for widespread use. As of 2025, MathML has over 95% global browser support, making it finally practical for production websites.
